Build Customization

Read the Docs supports customizing builds through a variety of mechanisms. This guide covers the most common use cases for build customization.

Using build.commands

The most powerful way to customize your build is by using the build.commands configuration key in your .readthedocs.yaml file. This allows you to run arbitrary commands during the build process.

version: 2
build:
  os: ubuntu-22.04
  tools:
    python: "3.11"
  commands:
    - pip install -r requirements.txt
    - sphinx-build -b html docs/ _readthedocs/html/

Build environment

Every build runs in a fresh, isolated Docker container. The build environment includes common system packages and tools needed for documentation generation.

  • Operating System: Ubuntu 22.04 LTS
  • Python: Multiple versions available (3.8 - 3.12)
  • Node.js: Available for JavaScript-based documentation tools
  • Rust: Available for mdBook and similar tools

Build process overview

The build process follows these steps:

  • Clone the repository
  • Install system dependencies
  • Create a virtual environment
  • Install Python dependencies
  • Run the build command
  • Upload the build output

Customizing with .readthedocs.yaml

The .readthedocs.yaml configuration file in the root of your repository is the primary way to configure Read the Docs. It supports the following top-level keys:

version: 2

build:
  os: ubuntu-22.04
  tools:
    python: "3.11"
    nodejs: "18"

sphinx:
  configuration: docs/conf.py
  builder: html

python:
  install:
    - requirements: docs/requirements.txt

Extending the build

For advanced use cases, you can extend the build process by adding pre- and post-build commands. These commands run before or after the main documentation build step.

Troubleshooting builds

If your build fails, check the build log for error messages. Common issues include:

  • Missing dependencies in requirements file
  • Incorrect path to configuration file
  • Incompatible Python or tool versions
  • Syntax errors in .readthedocs.yaml

For more help, visit the Read the Docs support page or ask in our community forum.

Advanced configuration

Read the Docs provides several advanced configuration options for power users. These include custom build images, private dependency installation, and build notifications.
